# SecureCam Client Startup Guide

This guide will walk you through setting up your **SecureCam Client** app (`securecam_client`) to run automatically on a Raspberry Pi OS system, as well as how to manage it. We'll cover everything from making it executable to creating a startup service, monitoring it, and removing it if needed.

---

## 1️⃣ Make Your App Executable

1. Make sure your app file exists. For example:

```bash
/home/pi/securecam_client.pi
```

2. Give it executable permissions:

```bash
chmod +x /home/pi/securecam_client.pi
```

3. Test it manually:

```bash
/home/pi/securecam_client.pi
```

> If it runs successfully, you’re ready to automate it.

---

## 2️⃣ Create a systemd Service

We’ll make a **systemd service** so it runs automatically on startup.

1. Open a new service file:

```bash
sudo nano /etc/systemd/system/securecam_client.service
```

2. Paste the following (adjust paths and user if needed):

```ini
[Unit]
Description=SecureCam Client Service
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi
ExecStart=/home/pi/securecam_client.pi
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

3. Save and exit (Ctrl+O → Enter → Ctrl+X).

### Notes:

* `Restart=always` will make the client restart if it crashes.
* `WorkingDirectory` ensures the app runs in the correct folder.
* `User=pi` should match the user you want to run the app as.

---

## 3️⃣ Enable the Service

Reload systemd and enable your service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable securecam_client.service
sudo systemctl start securecam_client.service
```

Check that it’s running:

```bash
systemctl status securecam_client.service
```

You should see it marked **active (running)**.

---

## 4️⃣ Viewing Logs

To see what your client is doing:

```bash
journalctl -u securecam_client.service -f
```

> Press `Ctrl+C` to stop viewing the live logs.

---

## 5️⃣ Stopping or Killing the Service

### Stop cleanly:

```bash
sudo systemctl stop securecam_client.service
```

### Disable autostart:

```bash
sudo systemctl disable securecam_client.service
```

### Force kill (if stuck):

```bash
sudo systemctl kill securecam_client.service
```

Check status:

```bash
systemctl status securecam_client.service
```

---

## 6️⃣ Deleting the Service Completely

1. Stop and disable:

```bash
sudo systemctl stop securecam_client.service
sudo systemctl disable securecam_client.service
```

2. Delete the service file:

```bash
sudo rm /etc/systemd/system/securecam_client.service
```

3. Reload systemd:

```bash
sudo systemctl daemon-reload
sudo systemctl daemon-reexec
```

4. Verify removal:

```bash
systemctl status securecam_client.service
```

It should say **Unit securecam_client.service could not be found**.

---

## 7️⃣ Optional Tips

* **GUI / camera access**: If your app needs a camera, you might need:

```ini
SupplementaryGroups=video
```

in your service file.

* **Delayed startup**: If you want the service to start after some delay:

```ini
ExecStartPre=/bin/sleep 10
```

* **Logging**: Redirect stdout/stderr to a log file if you want:

```ini
StandardOutput=file:/home/pi/securecam.log
StandardError=file:/home/pi/securecam_error.log
```

* **Test before rebooting**: Always start manually first to catch issues.

---

Now, your **SecureCam Client** is fully set up to start automatically, can be monitored, stopped, or removed cleanly, and will reliably send your remote video feed to your central SecureCam DVR.
